RULES:
1) ONLY use commands the repository config actually supports. Never invent scripts, tools, or package managers.
2) Enforce OUT OF SCOPE above. check_commands = fast PR feedback only: lint, typecheck, format, unit/component tests, compile/build, IaC validate.
3) CRITICAL PATH RULE: Every setup_commands, check_commands, and dependency_audit.cmd entry MUST start with:
   - "cd ${WORKDIR} && " for repo root, OR
   - "cd ${WORKDIR}/<subdir> && " for subdirectories.
   The literal substring ${WORKDIR} is replaced at runtime. Never emit bare commands without this prefix.
4) setup_commands: dependency installation for every stack that needs it before checks.
5) check_commands: for EACH detected stack subdirectory, include every configured dimension from the table (format, lint, typecheck, unit test, build) — typically 3–8 commands per stack, more in monorepos. Never skip format/lint when the repo has a script or tool config for it. Use confidence:
   - high: explicitly defined in repo config (e.g. package.json scripts.lint).
   - medium: tool clearly configured but no dedicated script.
   - low: uncertain (pipeline may skip low-confidence checks).
6) dependency_audit: one command appropriate for this repo's primary package manager, or null if not applicable / review_type is security-only context. Examples: npm audit, pip-audit, govulncheck, cargo audit — only when that ecosystem is present.
